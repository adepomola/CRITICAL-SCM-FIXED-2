# CRIT# Critical Thinking Project: Source Code Management for a Distributed Development Team

## Project Overview

This project demonstrates the implementation of Git as a distributed version control system for a software development team. It covers repository creation, branching, merging, conflict resolution, pull requests, and documentation using GitHub.

## Objectives

- Create a GitHub repository.
- Clone the repository to a local machine.
- Create and manage branches.
- Make commits with meaningful messages.
- Merge branches into the main branch.
- Resolve merge conflicts.
- Push changes to GitHub.
- Document the entire workflow in Markdown.
